# easy-scratch プログラミング講義カリキュラム案

## 講座タイトル

初めてのプログラミングに挑戦

## 講座のねらい

子どもたちが「プログラミングは難しいもの」ではなく、「自分の考えを順番に並べると、絵やキャラクターが動く楽しい表現方法」だと感じられることを目標にします。

特に今回は、最初に紙と鉛筆で「同じことを手作業で何度も行う大変さ」を体験し、その後で「同じ作業を命令にして実行すると、速く・正確に・何度でもできる」ことを体験してもらいます。プログラミングを、コンピューターに難しいことをさせるものではなく、「めんどうな繰り返しを楽にする道具」として理解できる構成にします。

低学年は、命令を順番に並べる体験と、キャラクターをスタートからゴールまで動かす体験を中心にします。高学年は、順番・繰り返し・条件分岐を使って、簡単なゲームを作る内容にします。

## 使用予定ツール

### 1-3年生クラス

- 使用ツール: 自作のWeb教材
- 形式: iPadのブラウザで開くWebアプリ
- 理由: 学校iPadに新しいアプリをインストールしなくても実施でき、児童用iPadの制限に合わせて画面や操作を単純化できます。低学年向けには、文字入力をできるだけ使わず、「右へ1マス」「上へ1マス」「色をぬる」などの命令カードを並べる形式にします。
- 事前確認: 学校貸与iPadのブラウザで教材URLにアクセスできるか。Webサイト閲覧制限がある場合は、事前にURLの許可設定が必要です。

### 4-6年生クラス

- 使用ツール: 自作のWeb教材
- 形式: iPadのブラウザで開くWebアプリ
- 理由: 低学年と同じ教材基盤を使いつつ、高学年向けには、繰り返し、条件分岐、スコア、当たり判定などを使える発展モードを用意します。アカウント作成や外部サービスへのログインを不要にできるため、学校環境で実施しやすくなります。
- 事前確認: 学校貸与iPadのブラウザで教材URLにアクセスできるか。作品を端末内に保存できない場合は、最後にスクリーンショットで記録する運用にします。

## 全体設計

### 1-3年生クラスのテーマ

紙でたいへんだった作業を、命令カードで楽にしよう

### 4-6年生クラスのテーマ

紙でたいへんだった作業を、くり返し・条件で自動化しよう

## 1-3年生クラス: カリキュラム案

想定時間: 60分

| 時間 | 内容 | ねらい |
| --- | --- | --- |
| 0-5分 | あいさつ・今日作るものの紹介 | 完成イメージを持つ |
| 5-15分 | 紙の方眼ワークで手作業をする | 同じ作業を何度も行う大変さ、間違いやすさを体験する |
| 15-22分 | 紙に書いた作業を命令カードに置き換える | 手作業を「命令の並び」として見直す |
| 22-30分 | 自作教材の基本操作 | ステージ、キャラクター、スタート、ゴール、命令カードに慣れる |
| 30-45分 | 命令カードで同じ作業を実行する | 紙で大変だった作業が自動で動くことを体験する |
| 45-55分 | 別の道順づくり | 命令カードを入れ替えながら、スタートからゴールまでの進み方を工夫する |
| 55-60分 | 見せ合い・まとめ | 自分の工夫を言葉にし、プログラミングの基本を振り返る |

### 作るもの

「キャラクターがスタートから出発し、命令カードの順番通りに動いてゴールする」小さなステージ。

例:

- 方眼のステージで、キャラクターをスタートからゴールまで動かす
- 指定されたマスで色をぬる
- 紙に書いた矢印の順番を、Web教材の命令カードに置き換える

### 扱う概念

- 手作業と自動実行の違い
- 紙の作業を命令に置き換える
- 順番に命令を並べる
- スタートの合図
- キャラクターを動かす
- 試して直す

### 手作業体験の例

最初に、児童が紙の方眼ワークで手作業をします。画面はまだ使わず、あえて紙と鉛筆だけで行います。

例:

- 方眼紙のスタートからゴールまで、矢印を1つずつ書いて道順を作る
- 「右、右、上、上、右」のような同じ道順を、別のマスにも何度も書く
- 指定されたマスを1つずつ手で塗って、簡単なドット絵を完成させる
- 途中で間違えたら消しゴムで直す、または最初からやり直す

その後、紙に書いた作業を「右へ1マス」「上へ1マス」「色をぬる」などの命令カードに置き換えます。最後にWeb教材で同じ命令を並べ、スタートボタンを押すだけで一気に実行します。

ここで体験してもらうこと:

- 手で何度も書くと時間がかかる
- 手で何度も書くと途中で間違えやすい
- 命令にすると同じ作業を何度でも再現できる
- 少し直したいときも、命令の一部を変えればよい

### 難易度調整

- 早く進む児童: 色をぬる場所を増やす、より短い命令の並べ方を考える、別の道順を作る
- 苦戦する児童: スタートからゴールまでの短い道順だけで完成にする

## 4-6年生クラス: カリキュラム案

想定時間: 60分

| 時間 | 内容 | ねらい |
| --- | --- | --- |
| 0-5分 | あいさつ・今日作るゲームの紹介 | 完成イメージを持つ |
| 5-15分 | 紙の方眼ワークで手作業をする | 繰り返し作業の大変さ、間違いやすさを体験する |
| 15-22分 | 紙の作業を命令に分解する | 作業手順を順番・繰り返し・条件として整理する |
| 22-30分 | 自作教材の基本操作 | ステージ、キャラクター、命令、実行に慣れる |
| 30-40分 | 命令で同じ作業を実行する | 紙で大変だった作業が自動で動くことを体験する |
| 40-48分 | 繰り返しブロックで作業を短くする | 同じ命令をまとめる便利さを体験する |
| 48-55分 | ゴール・障害物・当たり判定を作る | 条件分岐を体験する |
| 55-60分 | 自由改造・発表・まとめ | 工夫した点と学んだ考え方を共有する |

### 作るもの

「キャラクターを操作してゴールを目指す」ミニゲーム。

例:

- 迷路を抜けてゴールするゲーム
- 落ちてくるアイテムを集めるゲーム
- 障害物をよけながら進むゲーム

### 扱う概念

- 手作業と自動化の違い
- 紙の作業を命令に分解する
- イベント: 旗が押されたとき、キーが押されたとき
- 順次処理: ブロックを上から順番に実行する
- 繰り返し: ずっと動かす
- 条件分岐: もし触れたら、もしゴールしたら
- 座標: 右・左・上・下に動く
- デバッグ: 思った通りに動かない原因を探す

### 手作業体験の例

最初に、児童が紙の方眼ワークで、あえて時間のかかる作業を手で行います。

例:

- 方眼紙の迷路に、スタートからゴールまでの道順を矢印で1つずつ書く
- 同じ形の模様を3回描く
- 10マス進むために、矢印を10個書く
- 「壁に当たったら戻る」「宝物を取ったら点数を増やす」というルールを、毎回自分で判断して記録する
- 間違えたら消して直す、または最初からやり直す

次に、同じ処理を命令として並べます。さらに、同じ命令が何度も出てくる部分を「10回くり返す」のようにまとめます。最後に「もしゴールに触れたらクリア」「もし障害物に触れたら戻る」のような条件を追加し、人間が毎回判断しなくてもゲームが進むようにします。

ここで体験してもらうこと:

- 長い作業ほど、手作業では時間がかかる
- 同じ作業は「くり返し」にすると短く書ける
- 毎回の判断は「条件」にすると自動化できる
- 修正するときは、全部やり直さずに命令の一部を直せる

### 難易度調整

- 早く進む児童: 宝の数、壁の配置、スコア条件、くり返し・もしの組み合わせを増やす
- 苦戦する児童: キャラクター移動とゴール判定だけで完成にする

## 必要な準備物

### 運営側にお願いしたい準備物

- 児童用iPad
  - 1人1台が理想です
  - 難しい場合は2人1台でも実施可能です
  - 当日までに充電済みの状態にしていただけると助かります
- Wi-Fi環境
  - 児童用iPadが同時に接続できる環境
  - 自作Web教材のURLにアクセスできる状態
- 投影環境
  - 講師の画面を映せるプロジェクター、モニター、電子黒板など
  - HDMI接続、Apple TV、AirPlay、その他ミラーリング方法のいずれか
- 紙のワークシート
  - 方眼紙、または方眼入りワークシート
  - 1-3年生用: スタート、ゴール、キャラクター、通り道が入った簡単な方眼ワーク
  - 4-6年生用: 迷路、障害物、アイテム、ゴールが入った方眼ワーク
  - 印刷が難しい場合は、講師側で持参することも可能です
- 筆記具
  - 鉛筆
  - 消しゴム
  - 色鉛筆またはカラーペン
- 机の配置
  - 紙ワークとiPad操作の両方を行うため、児童が紙を書ける机が必要です
  - 2人1組で相談しながら進められる配置でも問題ありません
- サポート人員
  - iPad操作や紙ワークの補助に入れる大人がいると進行しやすいです
  - 目安として、15-20人に1人程度の補助があると安心です

### 学校側に確認したいこと

- 児童用iPadのブラウザで自作教材のURLにアクセスできるか
- Wi-Fi環境で同時接続しても問題ないか
- Webサイトの閲覧制限、JavaScriptの実行制限、ファイル保存制限があるか
- iPadの充電状況、予備端末の有無
- 教室の投影環境。HDMI、Apple TV、ミラーリング、プロジェクターなど

- 紙のワークシートを事前印刷できるか、または講師側で印刷物を持参する必要があるか
- 児童が鉛筆、消しゴム、色鉛筆またはペンを使えるか
- 1クラスあたりの人数、講座時間、サポートに入れる大人の人数

### 講師側で準備するもの

- 自作Web教材
- 低学年向け・高学年向けの教材URL
- QRコード
- 当日の手順スライド
- 低学年向け・高学年向けの完成見本
- 紙の方眼ワークシートの原稿
- 紙の命令カードの原稿
- 途中まで作った予備データまたは操作手順書
- Wi-Fiやアプリ制限があった場合の代替ワーク
- 発表用の進行メモ

## 紙ワークシートの具体案

### 1-3年生向け

方眼紙にスタート、ゴール、キャラクター、通り道を印刷します。児童はまず、キャラクターをゴールまで動かすための矢印を1つずつ手書きします。

活動例:

- スタートからゴールまでの矢印を書く
- 同じ道順をもう一度、別の場所に書く
- 指定されたマスを1つずつ塗ってドット絵を作る
- できた手順を「右へ1マス」「上へ1マス」「色をぬる」の命令カードに置き換える

児童に伝えるポイント:

- 紙で何度も書くのはたいへん
- 間違えると消して直す必要がある
- 命令にしておくと、同じ動きを何度でも実行できる

### 4-6年生向け

方眼紙に迷路、障害物、アイテム、ゴールを印刷します。児童はまず、手作業で道順、得点、ルール判定を記録します。

活動例:

- 10マス進むために矢印を10個書く
- 同じ模様や同じ道順を複数回書く
- アイテムを取ったら点数を手で足す
- 壁に当たったらスタートへ戻る、といった判断を毎回手で記録する
- その後、同じ処理を「くり返し」「もし」の命令に置き換える

児童に伝えるポイント:

- 同じ命令を何度も書くより、くり返しにしたほうが短い
- 毎回の判断は、条件にすると自動でできる
- 手作業ではミスしやすい作業ほど、プログラムにすると楽になる

## 自作Web教材の想定仕様

### 共通仕様

- iPadのSafariまたは学校指定ブラウザで動作する
- アプリのインストール、ログイン、個人情報入力は不要
- タップ操作中心で、キーボード入力をほぼ使わない
- 紙で行った方眼ワークと同じ課題を、Web教材上でも再現できる
- 作品はサーバーに保存せず、必要に応じて端末のスクリーンショットで記録する
- URLアクセスだけで始められるようにする
- ネットワーク不安定時に備え、できるだけ軽量な静的Webアプリとして作る

### 1-3年生向けモード

- 紙の方眼ワークと同じステージを表示する
- 紙に書いた矢印や色ぬりを、命令カードとして並べられる
- 命令カードをタップして順番に並べる
- 「スタート」を押すとキャラクターが命令通りに動く
- 使える命令は少数に絞る
  - 右へ1マス
  - 左へ1マス
  - 上へ1マス
  - 下へ1マス
  - 色をぬる
  - おはなし
- スタートとゴールが分かりやすく表示される
- 完成形は「スタートからゴールまで動く小さなステージ」にする

### 4-6年生向けモード

- 紙の方眼ワークと同じ迷路・模様・条件課題を表示する
- 紙で行った作業を命令として並べて再現する
- くり返し・条件モードでは、長い命令を短くしたり、自動で判定したりできるようにする
- 低学年向けの命令カードに加えて、発展ブロックを使える
  - くり返し
  - もしゴールに触れたら
  - もし障害物に触れたら
  - スコアを増やす
  - 宝を取ったら点を増やす
- 宝を集めながらゴールを目指すミニゲームを作れる
- 自由改造として、ステージ配置、キャラクター、ゴール、障害物、スコア条件を変えられる

## iPad・ネットワーク制限がある場合の代替案

### 自作Web教材にアクセスできない場合

低学年は、紙の命令カードを使ったアンプラグド活動に切り替えます。

例:

- 「右へ1マス」「上へ1マス」「色をぬる」「止まる」などのカードを並べる
- 友だちや講師がロボット役になり、カード通りに動く
- 最後にiPadのメモやKeynoteで簡単な絵コンテを作る

高学年は、アンプラグドの迷路ゲーム設計、またはiPadに入っているKeynote・メモ・Pagesなどを使い、ゲームのルール設計と画面設計を行います。

例:

- プレイヤー、ゴール、障害物、得点ルールを決める
- 「もし障害物に当たったらスタートに戻る」のように、条件分岐を言葉で書く
- 完成したゲーム画面を発表する

## 当日の進行方針

- 完成度よりも「自分で試して、直して、動いた」を重視する
- すべての児童が最低1つは動く作品を完成できるようにする
- 早く進む児童には自由改造を用意し、苦戦する児童には最小構成で完成できる道筋を用意する
- 発表は全員必須ではなく、希望者または数名の紹介にする
- 個人情報やアカウント作成が必要な進め方は避ける
